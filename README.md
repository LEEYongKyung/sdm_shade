# 서대문구 그늘막 설치 위치 선정 시뮬레이션

서대문구 내 그늘막 설치 후보지를 데이터 기반으로 평가하고, 지도와 표로 확인할 수 있는 업무용 웹 시스템입니다.

개발 의뢰 부서는 `재난안전과`이며, 기존에는 그늘막 설치 후보지를 선정하기 위해 약 7일 동안 민원 위치와 여러 자료를 수작업으로 검토했습니다. 이 프로젝트는 인도 폭, 도로, 교차로, 고령자 분포, 무더위쉼터, 기존 그늘막 위치를 종합해 보다 객관적인 설치 후보지를 제시하는 것을 목표로 합니다.

## 현재 구성

```text
.
├─ shade-priority-app/                         # React + Node.js + PostgreSQL 앱
│  ├─ src/                                     # 프론트엔드
│  ├─ server/                                  # Express API, DB, 평가 로직
│  ├─ public/                                  # favicon, OG 이미지
│  ├─ scripts/                                 # 배포/자료 생성 스크립트
│  └─ README.md                                # 개발 상세 문서
├─ 재난안전과_그늘막_설치위치_선정_설명페이지.html # 부서 설명용 정적 웹페이지
└─ 그늘막 설치 위치 선정 시뮬레이션 화면.png        # 설명페이지에 사용한 실제 화면 캡처
```

## 주요 기능

- 대로변 횡단보도를 기본 후보지로 사용
- 후보지별 종합점수 계산
- 선정 후보, 현장 확인 후보, 제외 후보 분류
- 지도에서 후보 위치와 평가 근거 확인
- 고령자 분포, 기존 그늘막, 무더위쉼터 보조 레이어 표시
- 점수체계 체크박스를 통한 평가 조건 시뮬레이션
- 후보 리스트와 기존 그늘막 리스트 엑셀 다운로드
- 연도별 설치 완료 그늘막 엑셀/CSV 업로드
- PostgreSQL/PostGIS 기반 데이터 저장

## 평가 기준

필수 조건은 총점에 포함하지 않고, 미충족 시 자동 제외합니다.

| 구분 | 항목 | 기준 |
| --- | --- | --- |
| 필수 | 인도 폭 3.5m 이상 | 국토지리정보원 인도 데이터 기준 |
가점 항목은 총 23점 만점입니다.

| 평가 항목 | 점수 |
| --- | ---: |
| 도로/간선도로성 | 0~5점 |
| 교차로/사거리 인접 | 0~5점 |
| 고령자 밀집 지역 | 0~6점 |
| 무더위쉼터 접근 부족 | 0~4점 |
| 기존 그늘막 위치 대비 거리 | 0~3점 |
| 합계 | 23점 |

기존 그늘막 위치 대비 거리 항목은 80m 이상 3점, 50~80m 2점, 30~50m 1점, 5~30m 0점으로 평가합니다. 기존 그늘막에서 5m 미만인 후보는 중복 설치 가능성이 높아 후보에서 제외합니다.

## 로컬 개발 실행

```bash
cd shade-priority-app
npm install
npm run dev
```

접속 주소:

- Frontend: `http://127.0.0.1:5173`
- Backend: `http://127.0.0.1:5174`

빌드 확인:

```bash
npm run build
```

상세 개발 문서는 [shade-priority-app/README.md](./shade-priority-app/README.md)를 참고합니다.

## 운영 서버 배포

현재 운영 방식은 NHN Cloud 인스턴스에서 다음 구조로 동작합니다.

- Nginx: 정적 프론트엔드 제공
- PM2: Node.js API 서버 실행
- PostgreSQL/PostGIS: 공간 데이터 저장
- API 내부 포트: `127.0.0.1:5174`

운영 서버에서 배포:

```bash
cd ~/Workspace/sdm_shade/shade-priority-app
bash scripts/deploy.sh
```

배포 스크립트가 수행하는 작업:

1. GitHub `main` 브랜치 최신 소스 pull
2. `npm install`
3. React 프론트엔드 빌드
4. `dist` 파일을 Nginx 배포 경로로 복사
5. PM2 API 서버 재시작
6. Nginx 설정 검사 및 reload
7. `/api/health` 상태 확인

운영 웹페이지:

```text
http://133.186.215.63/
```

## GitHub Pages로 설명페이지 공개하기

운영 시스템 전체가 아니라 설명용 정적 웹페이지만 GitHub Pages로 공개할 수 있습니다.

추천 구조:

```text
docs/
├─ index.html
└─ assets/
   ├─ favicon.png
   ├─ og-image.png
   └─ 그늘막 설치 위치 선정 시뮬레이션 화면.png
```

설정 순서:

1. 설명페이지 HTML을 `docs/index.html`로 복사
2. 설명페이지에서 사용하는 이미지를 `docs/assets/`로 복사
3. HTML 내부 이미지 경로를 `./assets/...` 형식으로 수정
4. GitHub 저장소 `Settings > Pages` 이동
5. `Build and deployment`에서 `Deploy from a branch` 선택
6. Branch는 `main`, Folder는 `/docs` 선택
7. 저장 후 배포 주소 확인

예상 주소:

```text
https://LEEYongKyung.github.io/sdm_shade/
```

주의:

- GitHub Pages는 공개 웹페이지로 노출될 수 있으므로 API Key, DB 비밀번호, 내부망 정보 등 민감정보를 포함하지 않습니다.
- 현재 설명페이지는 운영 시스템의 기능을 설명하는 정적 자료이며, 실제 후보지 평가 기능은 NHN Cloud 운영 서버에서 동작합니다.
