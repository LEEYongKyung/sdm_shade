# 서대문구 그늘막 설치 위치 선정 시뮬레이션

React + Node.js + PostgreSQL/PostGIS 기반의 그늘막 설치 후보지 평가 웹 앱입니다.

이 앱은 서대문구 내 그늘막 설치 후보지를 데이터 기반으로 선별하기 위한 업무용 도구입니다. 기존 PoC는 대로변 횡단보도, 교차로, 고령자, 무더위쉼터, 기존 그늘막 데이터를 활용했지만, `그늘막가이드.pdf`의 핵심 설치 기준인 **인도 폭 3.5m 이상** 조건을 충분히 반영하지 못했습니다. 새 버전은 보도대장 데이터를 추가해 현장 설치 실패율을 줄이는 것을 목표로 합니다.

## 현재 구현 상태

현재 앱은 서울시 OPENAPI를 실시간 호출하지 않고, 로컬 CSV/XLSX 파일을 기준으로 동작합니다.

| 데이터 | 현재 사용 방식 |
| --- | --- |
| 서울시 대로변 횡단보도 위치정보 | 로컬 CSV |
| 서울시 무더위쉼터 | 로컬 CSV |
| 서울시 교차로 관련 정보 | 로컬 CSV |
| 서울시 도로노선 정보 | 로컬 CSV |
| 서대문구 고령자 현황 | 로컬 CSV |
| 기존 그늘막 설치 위치 | 로컬 CSV |
| 보도대장/인도 폭 | 로컬 XLSX |
| 보도대장 주소 위경도 | VWorld API 호출 후 JSON 캐시 |
| 서울시 OPENAPI 수집 | 미구현, 후속 개발 대상 |

현재 로컬 데이터 기준 산정 결과:

| 구분 | 건수 |
| --- | ---: |
| 원천 횡단보도 후보 | 451 |
| 선정 후보 | 24 |
| 현장 확인 후보 | 133 |
| 제외 후보 | 294 |
| 기존 그늘막 | 110 |

## 제품 목표

- 서울시 대로변 횡단보도 위치정보를 기본 후보지로 사용한다.
- 각 후보지 주변의 인도 폭, 도로/간선도로성, 교차로, 고령자, 무더위쉼터, 기존 그늘막 정보를 매칭한다.
- 인도 폭 3.5m 미만 후보지는 자동 제외한다.
- 기존 그늘막 80m 이내 후보지는 자동 제외한다.
- 선정 후보, 현장 확인 후보, 제외 후보, 전체 후보, 기존 그늘막을 지도에서 확인한다.
- 점수체계 항목을 체크박스로 켜고 끄며 재산정한다.
- 매년 설치 완료 위치 엑셀/CSV를 업로드해 다음 평가에 반영한다.
- 데이터 매칭 실패 목록을 별도로 점검할 수 있게 한다.

## 화면 구성

### 지도 탭

지도 상단 탭:

- 선정 후보
- 현장 확인
- 제외 후보
- 전체
- 기존 그늘막

표시 방식:

- 선정 후보: 녹색
- 현장 확인 후보: 주황색
- 제외 후보: 빨간색
- 기존 그늘막: 검은색

`전체` 탭에서는 후보지 451건이 리스트에 표시되고, 지도에는 기존 그늘막 110건도 검은색으로 함께 표시됩니다. 기존 그늘막은 리스트에는 섞지 않습니다.

### 후보 리스트

리스트는 후보별 점수 근거를 함께 보여줍니다.

예시:

| 컬럼 | 예시 |
| --- | --- |
| 총점 | `15 / 20` |
| 보도폭 | `4m` |
| 도로/간선도로성 | `+5` |
| 교차로 | `25m (+5)` |
| 고령자 | `+6` |
| 쉼터 | `143m (+0)` |
| 상태 | 선정 / 현장 확인 / 제외 |

현재 구현은 일부 이전 점수체계를 사용하고 있으며, 아래의 20점 점수체계는 다음 개발 반영 대상입니다.

## 확정된 점수체계 기획

### 후보 생성 기준

후보지는 서울시 대로변 횡단보도 위치정보를 기본 단위로 생성합니다.

중요: `대로변 횡단보도 DB 일치`는 점수 항목으로 사용하지 않습니다. 모든 후보가 해당 DB에서 출발하므로 후보 간 우선순위를 가르는 기준이 되지 않기 때문입니다.

### 필수 조건

필수 조건은 총점에 포함하지 않습니다.

| 필수 조건 | 처리 |
| --- | --- |
| 인도 폭 3.5m 이상 | 미달 시 제외 |
| 기존 그늘막 80m 이내 | 해당 시 제외 |

### 20점 만점 점수 항목

| 평가항목 | 점수 |
| --- | ---: |
| 도로/간선도로성 | 0~5점 |
| 교차로/사거리 인접 | 0~5점 |
| 고령자 밀집 지역 | 0~6점 |
| 무더위쉼터 접근 부족 | 0~4점 |
| 합계 | 20점 |

### 도로/간선도로성, 5점

도로 규모와 도로 기능을 횡단보도 대기시간의 대리변수로 사용합니다.

| 조건 | 점수 |
| --- | ---: |
| 도로규모가 광로 또는 도로기능이 주간선도로 | +5 |
| 도로규모가 대로 또는 도로기능이 보조간선도로 | +4 |
| 도로규모가 중로 또는 도로폭 4m 이상 | +2 |
| 소로 또는 기타 도로 | +1 |
| 도로노선 매칭 실패 | 0 |

기존 PoC TOP100의 도로점수는 사용하지 않습니다.

### 교차로/사거리 인접, 5점

| 조건 | 점수 |
| --- | ---: |
| 50m 이내 | +5 |
| 50~100m | +4 |
| 100~150m | +2 |
| 150m 초과 | 0 |

### 고령자 밀집 지역, 6점

| 조건 | 점수 |
| --- | ---: |
| 고령자 비율 상위 20% | +6 |
| 상위 20~40% | +4 |
| 상위 40~60% | +2 |
| 그 외 | 0 |

### 무더위쉼터 접근 부족, 4점

| 조건 | 점수 |
| --- | ---: |
| 500m 이내 쉼터 없음 | +4 |
| 300~500m 이내 | +2 |
| 300m 이내 | 0 |

## 데이터 점검 기획

다음 데이터는 별도 `데이터 점검` 탭 또는 엑셀 다운로드로 확인할 수 있게 할 예정입니다.

| 점검 항목 | 의미 |
| --- | --- |
| 보도폭 매칭 실패 후보 | 후보지 주변 80m 이내 보도대장 구간을 찾지 못한 후보 |
| 도로노선 매칭 실패 후보 | 보도대장 노선명과 서울시 도로노선 정보가 연결되지 않은 후보 |
| VWorld 지오코딩 실패 주소 | 보도대장 주소 중 좌표를 찾지 못한 주소 |
| 좌표 이상 기존 그늘막 | 서대문구 범위를 벗어난 기존 그늘막 좌표 |
| 현장 확인 후보 전체 | 자동 제외는 아니지만 데이터 확인이 필요한 후보 |

데이터 점검 결과는 화면에서 조회하고, 엑셀로도 내려받을 수 있게 하는 것이 목표입니다.

## API와 로컬 파일 비교 필요성

현재 평가는 로컬 CSV/XLSX를 기준으로 수행됩니다. 운영 신뢰도를 높이려면 서울시 OPENAPI 응답과 로컬 파일이 같은지 비교해야 합니다.

비교 스크립트 기획:

```bash
npm.cmd run data:compare
```

비교할 항목:

- API 전체 건수와 로컬 파일 건수
- 주요 ID 기준 누락 데이터
- 좌표 컬럼 존재 여부
- 좌표계 차이
- 서대문구 필터 후 건수
- API에만 있는 행
- 로컬 파일에만 있는 행

예상 결과 형식:

| 데이터 | 로컬 건수 | API 건수 | 로컬에만 있음 | API에만 있음 | 판정 |

| 횡단보도 | 451 | 확인 필요 | 확인 필요 | 확인 필요 | 미검증 |
| 무더위쉼터 | 400 | 확인 필요 | 확인 필요 | 확인 필요 | 미검증 |

## 보도대장 지오코딩

보도대장은 주소 범위 기반이므로 VWorld 주소 API로 시작 주소와 끝 주소를 각각 지오코딩합니다.

현재 캐시 파일:

```text
server/data/sidewalk-geocodes.json
```

실행:

```bash
npm.cmd run geocode:sidewalks
```

현재 캐시 결과:

| 구분 | 건수 |
| --- | ---: |
| 전체 주소 | 364 |
| 성공 | 297 |
| 미검색 | 67 |
| 오류 | 0 |

미검색 주소는 주로 지번 일부, 교량, 구경계, 고가 하부, 학교명처럼 주소검색 API가 명확한 좌표를 반환하지 못한 항목입니다.

## PostgreSQL/PostGIS 운영 전환

스키마:

```text
server/db/schema.sql
```

초기화:

```bash
npm.cmd run db:migrate
npm.cmd run db:import
```

현재는 PostgreSQL이 없어도 로컬 데이터 모드로 실행됩니다. 운영 환경에서는 PostgreSQL/PostGIS로 전환하는 것이 좋습니다.

## 주요 API

### `GET /api/health`

서버 상태와 데이터 건수를 반환합니다.

### `GET /api/rules`

점수체계 목록을 반환합니다.

### `GET /api/candidates`

후보지를 재산정해 반환합니다.

Query:

- `mode`: `selected`, `review`, `excluded`, `all`
- `limit`: 반환 건수
- `enabled`: 적용할 점수체계 ID 목록

### `GET /api/existing-shades`

기존 그늘막 위치 목록을 반환합니다.

### `POST /api/uploads/installed-shades`

연도별 설치 완료 그늘막 엑셀/CSV를 업로드합니다.

### `GET /api/export/candidates.csv`

후보지 평가 결과를 CSV로 내려받습니다.

## 현장 확인 항목

다음 항목은 자동 점수화보다 현장 확인 체크리스트로 관리하는 것이 적절합니다.

- 점자블록과 60cm 이상 이격 가능 여부
- 음향신호기 인접 여부
- 차량 운전자와 보행자의 시야 방해 여부
- 지하 매설물 존재 여부
- 상가 간판 또는 시설물 간섭 여부
- 건물 그늘로 인한 실효성 저하 여부
- 실제 보도 유효폭과 보도대장 폭의 차이

- 확정된 20점 점수체계 코드 반영

- PostgreSQL/PostGIS 운영 모드 완성
